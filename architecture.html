The Big Picture

XForce Terminal is actually three separate projects that work together to create a complete trading platform. Think of it like a restaurant: you have the dining room (the terminal), the kitchen (the contracts), and the news feed (crypto info) - they all need to work together to give you a great experience.

ðŸ’¡ In Plain English

The Trading Terminal is what you see and use - it's the desktop app where you trade. The Smart Contracts are special programs on the Solana blockchain that help execute multiple trades at once to save you money. The News Service collects crypto news from around the web and feeds it to your terminal so you can stay informed while trading.

Trading Terminal - The main application you use

What Each Part Does

  • XForce Terminal - This is the main app you download and use. It has a beautiful desktop interface (like Bloomberg terminals), a backend server that handles your requests, and a web wallet helper. It's where you see prices, make trades, and manage your portfolio.
  • XForce Terminal Contracts - These are special programs that live on the Solana blockchain. Their main job is to let you do multiple trades in one transaction, which saves you a lot of money on fees. Instead of paying fees 5 times, you pay once!
  • XForce Crypto Info - This is like a news aggregator. It constantly scrapes the web for crypto news, analyzes whether articles are positive or negative, and serves them to your terminal so you can make informed trading decisions.

The Trading Terminal

The terminal is the heart of the system - it's what you interact with. Think of it as having three main parts: the beautiful desktop app you see, the backend server that does the heavy lifting, and a web helper for connecting wallets.

ðŸŽ¯ Why This Matters

Most trading tools are web-based, which means they're slower and less secure. By building a native desktop app, we get Bloomberg-terminal-level performance. Your private keys never leave your computer, and everything happens instantly. Plus, the modular design means other developers can reuse our code to build their own tools.

The Three Main Parts

1. Desktop App (What You See)

This is the beautiful Bloomberg-style interface that runs on your computer. It shows you real-time prices, lets you make trades, displays charts, and keeps track of your portfolio. It's built with Rust for maximum speed and security.

  • What it does: Shows you everything in a professional trading interface
  • Why it's special: Native desktop app means it's super fast - updates happen in less than 50 milliseconds
  • Where it lives: terminal/ folder

2. Backend Server (The Worker)

This is the server that runs in the background. When you click "swap tokens" in the desktop app, it's the backend that talks to Jupiter to find the best price, builds the transaction, and sends it to Solana. It also stores your trading history and handles authentication.

  • What it does: Handles all the complex work behind the scenes
  • Why it's special: Built with Axum (a super fast Rust web framework) - can handle thousands of requests per second
  • Where it lives: backend/ folder, runs on port 3001

3. Web Wallet Helper (Browser Tool)

This is a small web app that helps you connect popular wallets like Phantom, Solflare, or Backpack. It runs in your browser and makes it easy to link your existing wallet to the terminal.

  • What it does: Makes wallet connection simple and secure
  • Why it's special: Works with all major Solana wallets
  • Where it lives: wallet-web/ folder

The Smart Contracts

Smart contracts are programs that live on the Solana blockchain. Our main contract is called the "Batch Swap Router" and it does something really clever: it lets you do multiple trades in a single transaction, which saves you a ton of money.

ðŸ’¡ In Plain English

Normally, if you want to swap SOL â†’ USDC â†’ BONK â†’ RAY, you'd need to do three separate transactions and pay fees three times. Our batch swap contract lets you do all three swaps in ONE transaction, so you only pay fees once. This can save you 60-90% on transaction costs!

How It Works

The batch swap router is a program that's been deployed to the Solana blockchain. When you want to do multiple swaps, the terminal uses a special library (the "client") to build a transaction that calls this program. The program then executes all your swaps at once, atomically - meaning either they all succeed or they all fail. No partial swaps!

ðŸŽ¯ Why This Matters

Solana transaction fees are already super low (like $0.00025), but if you're doing complex trading strategies with multiple swaps, those fees add up. By batching swaps, you can execute sophisticated trading strategies for the cost of a single transaction. This makes advanced trading accessible to everyone, not just whales.

What the Contract Can Do

Feature What It Means Why It's Cool
Batch Swaps Do up to 10 swaps in one transaction Save 60-90% on fees compared to doing them separately
Atomic Execution All swaps succeed or all fail together No risk of partial execution - you're protected
Slippage Protection Won't execute if price moves too much Prevents bad trades from market volatility
Fee Management Calculates and tracks fees transparently You always know what you're paying

The Client Library

The client library is like a translator between the terminal and the smart contract. Instead of having to write complex blockchain code every time, developers can just use simple functions like "execute batch swap" and the library handles all the complicated stuff.

  • What it does: Makes it easy to interact with the smart contract
  • Why it's special: Type-safe - the compiler catches errors before you even run the code
  • Where it lives: xforce-terminal-contracts/client/
  • How terminal uses it: The terminal includes this library and calls its functions

The News Service

The news service is like having a personal assistant that reads crypto news 24/7 and summarizes it for you. It scrapes articles from all over the web, figures out if they're positive or negative, and feeds them to your terminal so you can make informed trading decisions.

ðŸ’¡ In Plain English

Imagine you're trading and want to know what's happening in crypto news. Instead of opening 10 different websites, the news service does that for you. It reads articles from CoinDesk, CoinTelegraph, RSS feeds, and more, figures out if each article is bullish or bearish, and shows you the most relevant news right in your trading terminal.

How News Gets to You

Step 1: Scraper Runs
Every 15 minutes, a Python script wakes up and visits crypto news websites. It reads RSS feeds, calls news APIs, and collects articles from CoinDesk, CoinTelegraph, and other sources.
Step 2: Data Gets Cleaned
The scraper uses Polars (a super fast data processing tool) to clean up the articles - removes duplicates, fixes formatting, and categorizes them (crypto, bitcoin, financial, etc.).
Step 3: Sentiment Analysis
Each article gets analyzed to determine if it's positive, negative, or neutral. This helps you quickly see if news is bullish or bearish for the market.
Step 4: Stored in Database
Everything gets saved to a PostgreSQL database - articles, sources, sentiment scores, and analytics. This database can hold millions of articles and search through them instantly.
Step 5: Terminal Requests News
When you open your terminal, it asks the news service "give me the latest crypto news" via an HTTP request.
Step 6: News Served to Terminal
The Rust API server quickly finds the latest articles, formats them nicely, and sends them back to your terminal.
Step 7: You See It in Your Terminal
The news appears in your trading interface, so you can see what's happening in crypto while you trade. No need to switch between apps!

The Three Parts of the News Service

1. Python Scraper (The Collector)

This is a Python script that runs continuously. It visits news websites, reads RSS feeds, calls APIs, and collects articles. It's like having a robot that reads crypto news 24/7 so you don't have to.

  • What it does: Collects news from multiple sources automatically
  • How often: Runs every 15 minutes (you can change this)
  • Where it lives: news-scraper/ folder

2. PostgreSQL Database (The Storage)

This is where all the news gets stored. It's a powerful database that can hold millions of articles and search through them in milliseconds. It also tracks analytics like which sources are most popular, what times of day news breaks, etc.

  • What it stores: Articles, sources, sentiment scores, analytics
  • Why PostgreSQL: Super fast, reliable, handles huge amounts of data
  • Where it lives: Runs as a separate database server

3. Rust API Server (The Server)

This is a web server built with Rust that serves news to your terminal. When your terminal asks for news, this server quickly finds the relevant articles and sends them back. It's super fast because it's written in Rust.

  • What it does: Serves news articles via REST API
  • How fast: Can handle thousands of requests per second
  • Where it lives: news-service/ folder, runs on port 3003

How Everything Connects

Now let's talk about how these three projects actually talk to each other. Understanding these connections helps you see how the whole system works together seamlessly.

ðŸ”— Terminal â†” Contracts: Using the Library

How they connect: The terminal includes the contracts client library directly in its code. It's like having a toolbox built right into the terminal - when you need to interact with the smart contract, the tools are already there.

What this means: When you click "execute batch swap" in the terminal, it uses code from the contracts library to build the transaction. The library knows exactly how to talk to the smart contract because it was designed specifically for it.

  • Type of connection: Code library (compiled into the terminal)
  • When it happens: When you build/compile the terminal
  • Why it's good: Super fast, no network calls needed, type-safe (catches errors before running)

ðŸŒ Terminal â†” News Service: HTTP Requests

How they connect: The terminal makes HTTP requests (like visiting a website) to the news service. It's like asking a friend "what's the latest news?" and they tell you.

What this means: When you open your terminal, it sends a request to the news service asking for the latest articles. The news service responds with a list of articles, and your terminal displays them. This happens over the network, so both services can run on different computers.

  • Type of connection: HTTP REST API (like a website API)
  • When it happens: When you use the terminal (runtime)
  • Where it connects: News service runs on port 3003
  • Why it's good: Flexible - news service can be on a different computer, easy to update independently

â›“ï¸ Terminal â†” Solana Blockchain: Direct Communication

How they connect: The terminal talks directly to the Solana blockchain using RPC (Remote Procedure Call) endpoints. Think of it like making a phone call to Solana - you ask questions and get answers.

What this means: When you want to check your balance, submit a transaction, or query account data, the terminal sends a request to a Solana RPC node (like Helius or a public endpoint). The node processes your request and sends back the answer.

  • Type of connection: RPC calls (like API calls but for blockchain)
  • When it happens: Every time you interact with the blockchain
  • What it does: Submits transactions, queries balances, gets account info
  • Why it's good: Direct connection means fast responses, you're talking to the source

How they connect: The smart contract is actually deployed (uploaded) to the Solana blockchain. It's like publishing a book - once it's on the blockchain, anyone can use it, and it stays there forever.

What this means: The batch swap router program lives on Solana at a specific address (the Program ID). When you send a transaction that calls this program, Solana validators execute it. The program has its own account on the blockchain where it stores data.

  • Type of connection: Deployed program (lives on blockchain)
  • Program ID: HS63bw1V1qTM5uWf92q3uaFdqogrc4SN9qUJSR8aqBMx
  • What it does: Executes batch swaps when called by transactions
  • Why it's good: Immutable (can't be changed), transparent, trustless

ðŸ”„ Terminal â†” Jupiter: Price Finding Service

How they connect: The terminal asks Jupiter (a DEX aggregator) for the best swap prices. Jupiter checks all the different exchanges on Solana and tells you which one has the best deal.

What this means: When you want to swap tokens, the terminal doesn't just pick a random exchange. It asks Jupiter "where can I get the best price for this swap?" and Jupiter responds with the optimal route, which might involve multiple exchanges to get you the best deal.

  • Type of connection: HTTP API calls

What Happens When You Do Things

Let's walk through exactly what happens when you perform common actions. This shows you how all the pieces work together in real-time.

When You Execute a Swap

This is the most complex flow - let's see what happens step by step when you swap tokens:

1 You Click "Swap"
In the desktop app, you select SOL and USDC, enter an amount, and click the swap button. The app immediately shows you a loading indicator.
2 App Asks Backend for Quote
The desktop app sends a request to the backend server: "Hey, what would I get if I swap 1 SOL for USDC?" The backend needs to find the best price.
3 Backend Asks Jupiter
The backend server asks Jupiter: "What's the best price for SOL â†’ USDC?" Jupiter checks all the exchanges (Raydium, Orca, etc.) and responds with the best route and expected output amount.
4 Backend Builds Transaction
The backend uses the contracts client library to build a transaction. It includes the Jupiter swap instructions AND a call to our batch swap contract to track and validate everything.
5 Transaction Sent to You
The backend sends the unsigned transaction back to your desktop app. It's like a form that needs your signature before it can be submitted.
6 You Review and Approve
The app shows you the details: "Swap 1 SOL for ~150 USDC, slippage 0.5%". You review it and click "Confirm". This is your chance to make sure everything looks right.
7 Transaction Gets Signed
Your wallet signs the transaction using your private key. This happens 100% on your computer - your private key never leaves your device. The signature proves you authorized this transaction.
8 Signed Transaction Sent to Backend
The signed transaction is sent back to the backend server. It's now ready to be submitted to Solana.
9 Backend Submits to Solana
The backend sends the transaction to a Solana RPC node. The node broadcasts it to the network, and validators start processing it.
10 Smart Contract Executes
Solana validators execute the transaction. The batch swap contract validates everything, Jupiter executes the swap on the best exchange, and your tokens are swapped. All of this happens in less than a second!
11 Success! Transaction Recorded
The backend saves your swap to the local database so you can see it in your transaction history. The app updates to show your new balances. You're done!

ðŸ’¡ In Plain English

Think of it like ordering food delivery: You place an order (click swap), the restaurant checks what's available (Jupiter finds best price), they prepare it (build transaction), you sign the receipt (sign transaction), they deliver (submit to Solana), and you get your food (tokens swapped). The whole process takes seconds, not minutes!

When You Use Batch Swaps

Batch swaps are special - they let you do multiple trades at once. Here's how that works:

1 You Plan Multiple Swaps
You want to do: SOL â†’ USDC â†’ BONK â†’ RAY. Instead of doing three separate transactions, you set up a batch swap.
2 Terminal Uses Contracts Library
The terminal uses the contracts client library to build a special transaction. This library knows how to talk to the batch swap contract.
3 Gets Quotes for Each Swap
For each swap in your batch, the terminal asks Jupiter for the best price. It collects all the quotes.
4 Builds Batch Transaction
The library builds one transaction that includes all three swaps plus a call to the batch swap contract. This is the magic - one transaction, multiple swaps!
5 You Sign Once
You sign the single transaction (not three separate ones). This saves you time and money.
6 Submitted to Solana
The transaction goes to Solana. Validators see it's a batch swap and process all swaps together.
7 Contract Validates Everything
The batch swap contract checks that all swaps are valid, amounts are correct, and slippage is acceptable. If anything is wrong, the whole transaction fails (protecting you).
8 All Swaps Execute Together
If validation passes, all three swaps execute atomically - they all succeed or all fail. No partial execution!
9 You Saved Money!
Instead of paying transaction fees three times, you paid once. You saved 66% on fees! Plus, all swaps happened instantly, so you didn't risk price changes between swaps.

ðŸŽ¯ Why This Matters

Batch swaps enable complex trading strategies that would be too expensive otherwise. Want to rebalance your portfolio across 5 tokens? Without batch swaps, that's 4 transactions and 4x the fees. With batch swaps, it's one transaction and one fee. This makes sophisticated trading accessible to everyone, not just people with huge amounts of capital.

Technologies Used

Here's a simple breakdown of all the technologies we use and why we chose them. Don't worry if you don't know what some of these are - the important thing is understanding what each project uses.

XForce Terminal

  • Rust - The programming language (super fast and safe)
  • egui - Makes the desktop interface
  • Axum - The web server framework
  • Leptos - For the web wallet (compiles to web code)
  • SQLite - Local database (stores your data)
  • Tokio - Handles async operations (doing multiple things at once)
  • Solana SDK - Talks to the Solana blockchain
  • JWT - For secure login
  • Braid Protocol - For messaging (CRDT-based)
  • rust-genai - AI assistant in the terminal

Smart Contracts

  • Rust - Same language as terminal (consistency!)
  • Anchor - Framework for building Solana programs
  • Solana Program Framework - The base layer
  • anchor-client - Library for interacting with contracts

News Service

  • Rust + Axum - Fast API server
  • PostgreSQL - Powerful database
  • Python - For the scraper (great for web scraping)
  • Polars - Super fast data processing
  • Leptos - Web frontend for browsing news

External Services

  • Jupiter - Finds best swap prices
  • Pyth - Provides price oracle data
  • Solana RPC - Helius or public endpoints
  • News APIs - NewsAPI, CryptoCompare, RSS feeds

ðŸ’¡ Why Rust?

Almost everything is written in Rust because it's incredibly fast (as fast as C++), memory-safe (catches bugs before they happen), and perfect for systems programming. When you're dealing with money and blockchain transactions, you want a language that won't let you make mistakes. Rust's compiler is like a very strict teacher that catches errors before your code even runs.

Development Tools

To build and work with these projects, we use:

  • Cargo - Rust's build system and package manager (like npm for JavaScript or pip for Python)
  • Git - Version control (tracks all changes to the code)
  • Anchor - Specialized tools for Solana program development
  • SQLx - Database toolkit that checks your SQL queries at compile time
